# VBoxWeb Provider

The VBoxWeb provider allows Terraform to manage [VirtualBox](https://www.virtualbox.org/) virtual machines through the `vboxwebsrv` SOAP web service API.

This provider supports **VirtualBox 7.1+** and enables infrastructure-as-code workflows for local virtualization environments, development workstations, and lab setups.

## Features

- **Clone VMs**: Create new virtual machines by cloning existing templates
- **Power Management**: Start and stop VMs as part of your Terraform workflow
- **Full Lifecycle**: Automatic cleanup of VMs and attached media on destroy

## Prerequisites

1. **VirtualBox 7.1+** installed
2. **vboxwebsrv** running and accessible

### Starting vboxwebsrv

```bash
# Without authentication (for testing/development)
vboxwebsrv --host localhost --authentication null

# With default authentication
vboxwebsrv --host localhost
```

## Example Usage

```hcl
terraform {
  required_providers {
    vboxweb = {
      source = "aslafy-z/vboxweb"
    }
  }
}

provider "vboxweb" {
  endpoint = "http://localhost:18083/"
  username = ""
  password = ""
}

resource "vboxweb_machine" "example" {
  name   = "my-vm"
  source = "ubuntu-template"
  state  = "started"
}
```

## Authentication

The provider connects to `vboxwebsrv` using SOAP over HTTP. Authentication depends on how `vboxwebsrv` is configured:

- **No authentication**: Start vboxwebsrv with `--authentication null` and leave username/password empty
- **Default authentication**: Uses VirtualBox's built-in authentication; provide valid credentials
